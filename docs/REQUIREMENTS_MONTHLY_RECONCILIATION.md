# 月度对账单需求说明（当前落地版）

更新时间：2026-03-24

## 1. 业务目标

在网页端上传两份 Excel：

1. 出入库记录（`.xlsx`）
2. 供应商账单（`.xlsx`）

系统自动完成：

- 出入库预处理（新增列、筛选类型）
- 账单预处理（同供应商+同采购单号+同 SKU 先汇总）
- 两侧按键关联并计算差异
- 生成可下载结果文件
- 保存每次任务记录（含原始上传和结果文件）

## 2. 输入文件约束

### 2.1 出入库记录（xlsx）

必需列：

- `变动`
- `入库价`
- `出入库类型`
- `来源单号`
- `sku`

处理规则：

1. 新增 `入库总价 = 变动 * 入库价`
2. 仅保留 `出入库类型` 属于：`采购入库`、`退货出库`
3. 数值统一保留两位小数

### 2.2 供应商账单（xlsx）

必需列：

- `采购单号`
- `SKU`
- `数量`
- `合计金额`

可选列：

- `供应商名称`
- `日期`
- `含运税价`

预处理规则（关键）：

- 按 `供应商名称 + 采购单号 + SKU` 分组（若无 `供应商名称`，按 `采购单号 + SKU`）
- 组内：
  - `数量` 求和
  - `合计金额` 求和
  - 新增 `账单汇总条数`（组内原始行数）

## 3. 对账匹配规则

关联键：

- 账单：`采购单号 + SKU`
- 出入库：`来源单号 + sku`

汇总后输出列：

- `出入库变动合计`
- `出入库入库总价合计`
- `数量差异 = 出入库变动合计 - 数量`
- `金额差异 = 出入库入库总价合计 - 合计金额`

数值规则：

- 全局数值列保留 2 位小数
- 差异列把极小浮点残差归零（避免 `E-13`）

## 4. 结果文件与历史记录

结果文件：`reconciliation_result.xlsx`

工作表：

1. `账单核对`
2. `出入库明细_筛选后`

任务持久化目录：`data/jobs/<job_id>/`

保存内容：

- 原始上传出入库文件
- 原始上传账单文件
- 结果文件
- `job_meta.json`（含 `created_at`、文件名、摘要）

前台能力：

- 查看历史记录（时间倒序）
- 下载当次：结果 / 出入库 / 账单

## 5. 接口清单

- `POST /api/monthly-statement/run`
- `GET /api/monthly-statement/download/{job_id}`
- `GET /api/monthly-statement/runs`
- `GET /api/monthly-statement/runs/{job_id}/download/inout`
- `GET /api/monthly-statement/runs/{job_id}/download/supplier`

## 6. 已确认口径

- 同 SKU 但不同采购单号：不合并
- 仅在 `供应商 + 采购单号 + SKU` 完全一致时才先汇总账单
